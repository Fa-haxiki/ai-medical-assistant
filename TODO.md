## 项目 TODO 清单（AI 医疗助手）

> 本清单按后端、前端、RAG/知识库、工程与体验四个维度整理，方便逐步迭代。

### 一、后端（Node / RAG / 多模态）

- [ ] **对话历史持久化**
  - 将 `backend-node/src/store.ts` 的内存存储迁移到可持久化方案（例如 SQLite/PostgreSQL/Redis），支持进程重启后恢复会话。
  - 为会话增加简单的过期/清理策略（如按时间或数量清理历史）。

- [ ] **请求与错误日志优化**
  - 为关键接口（`/api/chat*`、`/api/knowledge/upload` 等）增加结构化日志（包含会话 ID、RAG 命中情况、耗时）。
  - 区分用户错误（4xx）与系统错误（5xx），统一错误响应格式。

- [ ] **参数与限流校验**
  - 对入参进行更严格校验（消息长度、图片大小/分辨率、上传文件大小上限等）。
  - 增加基础限流（按 IP / 会话 ID）与并发保护，避免误操作打爆后端或百炼额度。

- [ ] **多模态与文生图健壮性**
  - 针对 DashScope/百炼返回错误码做更细粒度处理与重试（网络抖动、超时、配额不足）。
  - 为多模态接口补充更清晰的错误信息（比如“图片过大/格式不支持”等）。

- [ ] **知识文件上传增强**
  - 为上传的文件增加元数据记录（文件名、上传时间、片段数），便于后续查看与管理。
  - 提供简单的「知识库列表/统计」接口（总文档数、总片段数、最近上传记录等）。
  - 支持按文件名或标签删除/重建部分向量（目前只能追加）。

- [ ] **配置与安全**
  - 检查所有敏感配置仅从 `.env` 读取，避免硬编码。
  - 为生产环境准备示例配置（日志等级、CORS 白名单、端口等）。

### 二、前端（聊天体验 / UI / 状态管理）

- [ ] **聊天输入组件化与复用**
  - 将当前 `ChatInterface` 中的输入区域进一步拆分为独立组件（文本输入 + 图片上传），便于后续单独测试/复用。

- [ ] **加载与错误提示统一**
  - 统一 loading / 错误提示的交互：例如顶部 toast 或消息区域内的系统消息，附带重试按钮。
  - 对多模态和文生图请求的失败做更友好的提示（例如引导用户缩小图片或降低并发）。

- [ ] **对话管理能力**
  - 增加「新建会话」「历史会话列表」「切换会话」入口，而不仅依赖本地存储的单个 conversationId。
  - 为每个会话增加简短标题（首条问题摘要）方便用户识别。

- [ ] **移动端与无障碍优化**
  - 进一步优化小屏布局（键盘弹出时的滚动与输入区域可见性）。
  - 为主要交互元素补充 ARIA 标签和键盘操作支持。

### 三、RAG / 知识库相关

- [ ] **多文件与分层知识库管理**
  - 在 Chroma 中为不同文件或知识域设置 `metadata`（如 `category`、`version`），支持按类别检索。
  - 支持从前端按类别筛选或选择知识范围（例如仅使用“银屑病指南”或“用药说明”等）。

- [ ] **上传格式扩展与体验**
  - 为 PDF / Word 上传增加解析异常时的详细错误反馈，并提示用户转为 Markdown/纯文本。
  - 考虑支持 `.doc`、`.pptx` 等常见格式（通过额外解析库或异步预处理服务）。

- [ ] **RAG 质量可观测性**
  - 在后端日志中记录每次 RAG 检索到的文档标题/片段摘要，便于排查检索结果是否合理。
  - 可选：在前端提供一个「调试模式」，显示 RAG 命中的知识片段（仅开发环境开启）。

### 四、工程质量与运维

- [ ] **测试与类型检查**
  - 为关键后端模块（`chat.ts`、`rag.ts`、`routes/chat.ts`）添加单元测试或集成测试。
  - 为前端核心逻辑（流式解析、Markdown 渲染、多模态流程）添加至少一轮基础测试。

- [ ] **脚本与一键启动体验**
  - 在根目录提供统一的 `dev` 脚本（例如：同时启动 `backend-node`、`frontend`，并在需要时提醒用户启动 Chroma）。
  - 在 README 中补充「本地开发一键步骤」与「常见问题 FAQ」（例如百炼配额不足、CORS 报错、Chroma 未启动等）。

- [ ] **生产部署建议**
  - 在文档中增加推荐的生产部署方案：例如 Nginx 反向代理 + PM2 / Docker Compose（前后端 + Chroma）。
  - 提示在生产环境下开启 HTTPS、访问日志与基础监控。
